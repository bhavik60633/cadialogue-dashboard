# CADialogue Dashboard — Deployment Guide

Get the dashboard online at `https://dashboard.cadialogue.in` so your team
can log in daily and publish 5-10 articles each.

| Item                  | Where to get it                                                                                              |
| --------------------- | ------------------------------------------------------------------------------------------------------------ |
| **GitHub account**    | https://github.com/join (free)                                                                               |

## Step 1 — Create a GitHub repository

1. Go to https://github.com/new
2. **Repository name**: `cadialogue-dashboard`
3. **Private** (so your code stays internal)
4. Do **NOT** check "Add a README" — we already have one
5. Click **Create repository**
6. GitHub will show you a page with a URL like:
   `https://github.com/YOUR_USERNAME/cadialogue-dashboard.git`
   Copy this URL.

---

## Step 2 — Push the code from this PC

Open **PowerShell** in the `Prototype` folder, run these commands one at a time:

```powershell
cd "C:\Users\PC\OneDrive\Desktop\Prototype"

git init
git add .
git commit -m "Initial dashboard upload"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/cadialogue-dashboard.git
git push -u origin main
```

> If `git push` asks for credentials, use your GitHub username and a
> **Personal Access Token** (Settings → Developer settings → Personal access
> tokens → Tokens (classic) → Generate new token → check `repo` scope).
> Paste the token in place of your password.

Your code is now on GitHub. **Your `.env` is NOT pushed** — `.gitignore`
keeps it out. We'll add the keys to Render in the next step.

---

## Step 3 — Deploy to Render (one-click)

1. Sign in at https://dashboard.render.com (use **Sign in with GitHub**).
2. Click **New +** → **Blueprint**.
3. Select your `cadialogue-dashboard` repository.
4. Render reads `render.yaml` and shows the service to create.
   Click **Apply**.
5. Render starts building. This takes **5-8 minutes** the first time
   (installs Python, Node, builds Next.js, builds the Docker image).

While it builds, go to step 4 to add your API keys.

---

## Step 4 — Add your API keys to Render

Still in the Render dashboard:

1. Click your service `cadialogue-dashboard`.
2. Go to **Environment** tab.
3. For each key below, click the value field and paste the matching
   value from your local `.env`:

| Variable               | Value source                                                |

| `OPENAI_API_KEY`       | Your OpenAI key (`sk-svcacct-…`)                            |

| `PEXELS_API_KEY`       | Your Pexels key                                             |
| `NEWSAPI_KEY`          | Your NewsAPI key                                            |
| `WP_URL`               | `https://cadialogue.in`                                     |
| `WP_USERNAME`          | `renuka.malik99@gmail.com`                                  |
| `WP_APP_PASSWORD`      | `xxxx xxxx xxxx xxxx xxxx xxxx`                             |

`AUTH_SECRET` and `PIPELINE_SERVICE_TOKEN` are auto-generated by Render — leave them.

Click **Save Changes**. Render redeploys automatically (2 minutes).

## Step 5 — Add a custom domain (`dashboard.cadialogue.in`)

1. In Render → your service → **Settings** → **Custom Domain** → **Add Custom Domain**.
2. Enter `dashboard.cadialogue.in` → click **Save**.
3. Render shows you a CNAME target like `cadialogue-dashboard.onrender.com`.
4. Go to your **Hostinger control panel**:
   * Domains → `cadialogue.in` → **DNS / Nameservers**.
   * Add a **CNAME record**:
     * Name: `dashboard`
     * Target: the value Render gave you
     * TTL: default
5. DNS propagation takes 5-30 minutes. Render auto-issues an HTTPS certificate.

When done, https://dashboard.cadialogue.in opens your login page.

## Step 6 — Create team logins

The very first user account already exists locally
(`bhavikvaid65@gmail.com` / `Cadialogue@2026`) but won't be on Render
because state is fresh. To create accounts for your colleagues:

### Option A — SSH into Render and run the script
1. Render → your service → **Shell** tab → click **Launch Shell**.
2. Run:
   ```sh
   python -m pipeline.scripts.add_user colleague@email.com --password=ChooseAStrong! --role=editor
   ```
3. Repeat for each team member.

### Option B — Roles
* `--role=admin` — full access (use for yourself)
* `--role=editor` — can approve/generate/publish but cannot delete users

Tell your team to log in at https://dashboard.cadialogue.in/login.

## How your team uses it (daily workflow)

1. **Morning**: log in at https://dashboard.cadialogue.in.
2. Click **"Discover Today's Topics"** → 10 topics appear in ~40 seconds.
3. **Approve** the topics they like, **Reject** the rest.
4. Click **"Generate Article →"** on each approved topic.
5. After 1-2 minutes the article is ready. Click **"Add Images →"**.
6. In each article section: click **"Find real photos"** → pick a photo.
7. Click **Publish** → article goes live on cadialogue.in with images.
8. Refresh — the card turns green and shows the live URL.

5-10 articles per day is realistic. Each costs ~$0.10 in GPT-4o API spend.

## Troubleshooting

**"Application failed to start"** in Render logs
→ Check the **Logs** tab. Almost always a missing env var. Re-check Step 4.

**Articles still failing to publish**
→ Render Shell → `cat /data/state/last_generation_error.txt`
→ Full traceback shown.

**Free tier sleeping**
→ Upgrade to **Starter ($7/mo)** in Render → never sleeps, instant response.

## Security notes

- `.env` is never pushed (in `.gitignore`).
- All Render env vars are encrypted at rest.
- FastAPI binds to 127.0.0.1 inside the container — only the Next.js proxy can reach it.
- NextAuth uses bcrypt hashing for passwords + JWT sessions (8-hour expiry).
- HTTPS is automatic on Render.
- WordPress uses Application Passwords (not your main login).
